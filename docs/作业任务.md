## Web Data Applications 小组项目任务清单

以下任务依据《Assessment Brief - Group Project 2025-26》整理，供团队在 Student Bay 项目执行过程中逐项勾选与追踪。[来源](file:///d%3A/OneDrive/MyProjects/RegisterDemo/Assessment%20Brief%20-%20Group%20Project%202025-26.pdf)

### 1. 总体目标
- 设计与实现 Student Bay 校园二手交易平台，覆盖用户验证、物品发布、搜索发现与交易管理全流程。
- 完成前端、后端、Demo 视频与个人反思报告四类交付，提交至 UWE Blackboard（截至 2025-11-28 17:00 UK）。

### 2. 交付物结构
1. **代码仓库**
   - `front-end/`：`index.html`、CSS、JS、静态媒体。
   - `back-end/`：Express 服务器、数据库架构、配置文件。
   - 根目录 `README.md`：运行与测试说明。
2. **视频演示**：≤5 分钟，包含项目概览、已实现 FR、功能演示。
3. **个人反思报告**：500 词，涵盖角色分工、问题求解、协作体验与改进建议。

### 3. 必做功能需求
#### FR1 用户档案与验证
- 支持 Public / Student / Affiliated 三类成员，学生与教职需分别通过大学邮箱验证。
- 资料页展示：验证状态、加入日期、成功销售数、平均评分、资料完整度百分比（头像、简介、学校、年级等字段驱动）。

#### FR2 物品发布与展示
- 发布时上传 ≤5 张图片，填写描述、价格、状况、浏览计数。
- 按类别记录专属字段：
  - Textbooks：ISBN、Course Code、Module Name、Edition、Author。
  - Electronics：Brand、Model Number、Warranty Status、Original Purchase Date、Accessories Included。
  - Furniture：Item Type、Dimensions、Material、Assembly Required、Condition Details。
  - Clothing：Size、Brand、Material、Colour、Gender。
  - Sports Equipment：Brand、Size/Dimensions、Sport Type、Condition Details。

### 4. 自选功能（至少选择 1 组）
- **FR3 搜索与发现**：关键词、类别、价格、状况筛选；教材支持 Course Code / Module Name / ISBN 查询；结果展示缩略图、价格、卖家评分、发布日期、浏览量。
- **FR4 关注与通知**：关注列表、价格或状态变更通知、匹配条件新物品邮件。
- **FR5 状态管理**：Available / Reserved / Sold 流程；Sold 时记录买家（站内或站外）并触发评分流程。

### 5. 后端要求
- 使用 Node.js + Express 实现 REST API、路由、中间件、表单/JSON 处理、会话或 JWT。
- 数据库任选 SQLite 或 MongoDB，需具备 CRUD、结构化 schema、选择理由与文档。

### 6. 评分要点提醒
- **前端 (45%)**：HTML/CSS/JS 规范、响应式体验、需求覆盖度。
- **后端 (45%)**：API 完整性、路由/中间件设计、数据库结构与文档。
- **Demo (10%)**：结构清晰、音视频质量佳、完整演示功能与实施细节。
- **Reflection (10%)**：深度反思角色、策略、协作、收获与改进。

### 7. 合规性
- 提交 zip 打包所有交付件，遵守 UWE 学术诚信与生成式 AI 使用限制；仅可用 AI 辅助排障或语法纠正，并需在提交材料中声明。
